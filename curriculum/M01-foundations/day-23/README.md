# 🗓️ Day 22 — Branching & History

Today you'll learn Git branching strategies, merge techniques, rebasing, and history manipulation.

## Goal

Create and manage branches, resolve merge conflicts, rebase feature branches, and clean up commit history.

## Key Learnings

- Branch creation, switching, and deletion
- Fast-forward vs 3-way merges
- Merge conflict detection and resolution
- `git rebase` — linearizing history
- `git rebase -i` — interactive rebase (squash, reword, drop)
- `git log --graph` — visualizing history

## Why This Matters

Git history is your team's changelog, audit trail, and deployment artifact. Clean branching = safe deployments. Messy history = "did we deploy that hotfix?"

④ Theory · 5 min
📌 FHS — Chaar Groups Mein Samjho

The Linux Directory Tree

/ (root — filesystem ka entry point) ├── etc/ ← system-wide configs ├── var/ │ └── log/ ← all system & app logs ├── home/ │ └── username/ ← user personal data ├── root/ ← root user's home (NOT /)! ├── tmp/ ← temp (CLEARED on reboot) ├── proc/ ← virtual: kernel interface (RAM) ├── sys/ ← virtual: device tree (RAM) ├── bin/ → usr/bin ← essential binaries (ls, cat, grep) ├── usr/ ← most user-space software ├── opt/ ← optional third-party software └── lib/ → usr/lib ← shared libraries (.so files)

Note: green = virtual filesystem (RAM-based, not on disk)

① Group 1 — /etc, /var, /var/log

/etc
System-wide configuration files. nginx, ssh, fstab, passwd, hosts — sab yahan. Never for data or binaries! Read at boot or service start.
Examples: /etc/nginx/, /etc/ssh/sshd_config, /etc/fstab, /etc/hosts
/var
Variable data — changes at runtime. Logs, spool, databases, cache. Grows over time. Monitor disk usage here!
Examples: /var/log/, /var/lib/mysql/, /var/cache/, /var/spool/
⚠ /var/log — Production Alert!
/var/log holds ALL system and app logs: /var/log/syslog, auth.log, nginx/access.log. /var/log fills up → services crash → this IS a production incident! Set up disk alerts on /var separately.

② Group 2 — /home, /root, /tmp

/home/username
User's personal space. Dotfiles (.bashrc, .ssh/), documents, downloads. Each user gets /home/their_name.
/root
Root user's home directory. NOT the filesystem root /! Common confusion — / is root of FS, /root is root user's home.
/tmp
Temporary files. CLEARED on reboot! Never store important data here. Use /var/lib/appname for persistent app data.

③ Group 3 — /proc, /sys (Virtual Filesystems)

Virtual FS — RAM-based, Created Fresh at Boot

Boot → Kernel mounts /proc + Kernel mounts /sys → RAM only — no disk write
/proc
Kernel interface. Real-time process and system info. NOT on disk — exists only in RAM. Read files here to inspect running kernel state.
/proc/cpuinfo, /proc/meminfo
/proc/mounts, /proc/1/maps
/proc/sys/net/ipv4/ip_forward
/sys
sysfs — exposes kernel device tree. Used for tunables and hardware introspection. Write values here to tune kernel params at runtime.
/sys/class/net/ (NICs)
/sys/block/ (disks)
/sys/devices/ (hardware tree)

④ Group 4 — /bin, /usr, /opt, /lib

/bin → /usr/bin
Essential user binaries: ls, cat, grep, cp, mv. On modern systems /bin is a symlink to /usr/bin. Do NOT put custom scripts here.
/usr
Most user-space software: /usr/bin (all binaries), /usr/lib (libraries), /usr/share (docs, data). Think of it as "Unix System Resources".
/opt
Optional third-party software. Chef, Puppet, custom enterprise installs, vendor-packaged apps. Keeps third-party stuff out of /usr.
/lib → /usr/lib
Shared libraries (.so files). Like DLLs on Windows. Required by binaries in /bin and /sbin. On modern systems symlinked to /usr/lib.
Custom scripts kahan rakho? → /usr/local/bin/ for system-wide custom scripts. This is the correct FHS location — never pollute /usr/bin or /bin directly.

Step 1 — /etc Explore Karo

# /etc mein kya kya hai?
ls /etc

# OS info — distro aur version
cat /etc/os-release

# Hosts file — DNS override map
cat /etc/hosts

# Password file — all system users
cat /etc/passwd

# nginx config (agar installed hai)
ls /etc/nginx/
cat /etc/nginx/nginx.conf

# SSH server config
cat /etc/ssh/sshd_config | grep -v '^#' | grep -v '^$'

Step 2 — /var/log Explore Karo (Disk Check Bhi!)

# /var/log mein kaun kaun si log files hain?
ls -lh /var/log/

# syslog ke last 20 lines
tail -20 /var/log/syslog

# Auth log — login attempts
tail -20 /var/log/auth.log

# /var disk usage — CRITICAL to monitor!
df -h /var

# Biggest log files (agar disk bhar raha hai)
du -sh /var/log/* 2>/dev/null | sort -rh | head -10

Step 3 — /proc aur /sys Virtual Filesystems

# CPU info — virtual file, not on disk!
cat /proc/cpuinfo | grep "model name" | head -1

# Memory info
cat /proc/meminfo | grep -E "MemTotal|MemFree|MemAvailable"

# Currently mounted filesystems
cat /proc/mounts

# Network interfaces via sysfs
ls /sys/class/net/

# IP forwarding status (1=enabled, 0=disabled)
cat /proc/sys/net/ipv4/ip_forward

# Currently running processes — each PID has a folder
ls /proc/ | grep '^[0-9]' | head -5
cat /proc/1/cmdline && echo   # PID 1 = init/systemd command

Step 4 — Software Kahan Rehta Hai?

# which — binary ka path find karo
which nginx
which python3
which bash

# whereis — binary + man page + source
whereis nginx

# Symlink resolve karo — actual binary location
readlink -f $(which python3)

# /opt — third party software check
ls /opt/

# /usr/local/bin — custom scripts yahan rakho
ls /usr/local/bin/

# Shared libraries list
ls /usr/lib/ | head -10
ldd $(which ls)              # libraries ls needs
⑥ Common Mistakes · 5 min
⚠ Ye Galtiyan Mat Karna!
Golu
"Main /tmp mein app data store kar raha tha Jagu — server reboot ke baad sab gone!"
① App data /tmp mein store karna — reboot pe data gone!
mkdir /tmp/myapp-data && cp important.db /tmp/myapp-data/ # WRONG! Reboot = data lost!
mkdir -p /var/lib/myapp && cp important.db /var/lib/myapp/ # Correct — persistent storage ✓
② Custom script /usr/bin mein daalna — system upgrade pe overwrite ho sakta hai
cp my-deploy-script.sh /usr/bin/deploy # WRONG! /usr/bin is for OS packages only
cp my-deploy-script.sh /usr/local/bin/deploy && chmod +x /usr/local/bin/deploy # Correct ✓
③ /var/log disk usage monitor nahi karna — disk full = production outage
# No monitoring on /var/log — one day disk is 100% full, all services crash
# Set up disk alert on /var separately (not just /) df -h /var # Manual check # Production: alert when /var > 80% usage # Logrotate configured to rotate /var/log/* automatically ✓
⑦ Interview Questions · 5 min
❓ Interview Mein Yeh Poochenge
1
What is the difference between /bin and /usr/bin?
▾
Historically, /bin contained essential binaries needed for single-user recovery mode (ls, cat, mount, etc.) and was on a separate small partition. /usr/bin had the rest of user-space binaries and required /usr to be mounted. On modern Linux systems (Debian/Ubuntu/Fedora 17+), /bin is a symlink to /usr/bin — the distinction is largely historical. However, knowing this is expected in senior-level interviews. Safe answer: "/bin is essential binaries for system recovery; /usr/bin is the main location for user-space executables. On modern systems /bin symlinks to /usr/bin."
2
Where would you put a custom script that needs to be available system-wide?
▾
/usr/local/bin/ — this is the FHS-correct location for locally installed scripts and binaries that are not managed by the system's package manager. Never put custom scripts in /usr/bin (managed by package manager, could be overwritten on upgrades) or /bin (system essentials only). Place script there, chmod +x it, and it will be in every user's PATH automatically.
3
Why does /tmp get cleared on reboot?
▾
/tmp is designed for temporary files that processes need only during their current run. By clearing it on reboot, the OS ensures no stale lock files, socket files, or partial data from dead processes accumulate. On systemd systems this is handled by systemd-tmpfiles-clean.service and configured in /usr/lib/tmpfiles.d/. Important: some distros also clear /tmp on a timer (not just reboot). Never store anything important in /tmp — use /var/lib/appname for persistent application data.
4
What is the difference between /etc and /var?
▾
/etc contains static configuration — files that humans edit to configure the system. They change rarely and only when an admin makes intentional changes. Examples: /etc/nginx/nginx.conf, /etc/ssh/sshd_config, /etc/fstab.

/var contains variable/runtime data — files that change automatically as the system runs. Programs write here, not admins. Examples: /var/log (logs written by services), /var/lib/mysql (database files), /var/spool (mail/print queues), /var/cache. Key distinction: /etc should be in version control; /var should be in backups and disk monitoring.

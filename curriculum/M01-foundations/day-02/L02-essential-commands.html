③ Theory · 5 min
📚 The 4 Command Groups

🔙 Group 1 — Navigation

These commands answer two questions: Where am I? and What is here? Never skip these before running anything destructive.

CommandWhat it doesCommon flags
pwdPrint Working Directory — shows your exact current location-L (logical), -P (physical)
cd /pathChange Directory — move to a different locationcd ~ (home), cd - (previous dir), cd .. (up one level)
lsList directory contents-l (long format), -a (show hidden), -h (human readable sizes), -la (combined)
treeDisplay directory structure as a visual tree-L 2 (depth limit), -a (include hidden), -d (dirs only)
 
# Where am I right now?
pwd
# Output: /home/golu/devops-lab

# What is in this directory?
ls -la
# -l = long format (permissions, size, date)
# -a = show hidden files (dotfiles like .bashrc)

# Navigate and visualize
cd /etc
tree -L 2

📄 Group 2 — File Operations

Create, copy, move, delete. These are your hands in the filesystem. Most production disasters happen from misusing these — especially rm.

CommandWhat it doesKey flags
mkdir dirMake a new directory-p (create parent dirs, no error if exists)
touch fileCreate empty file or update timestamp-t (set specific timestamp)
cp src dstCopy file or directory-r (recursive for dirs), -i (interactive/prompt), -v (verbose)
mv src dstMove or rename file/directory-i (interactive), -v (verbose), -n (no overwrite)
rm fileRemove file or directory-r (recursive dirs), -f (force, no prompt), -i (interactive)
find /pathSearch for files by name, size, date, type-name, -size, -mtime, -type f/d
ln -s src dstCreate a symbolic (soft) link-s (symbolic), without -s creates hard link
 
# Create nested directories in one shot
mkdir -p ~/devops-lab/m00/u01/day02

# Create empty files
touch notes.txt config.yaml README.md

# Copy with interactive prompt (safer)
cp -i config.yaml config.yaml.bak

# Find files larger than 100MB
find / -size +100M -type f 2>/dev/null

# Create a symlink
ln -s /etc/nginx/nginx.conf ~/nginx.conf

📊 Group 3 — System Info

Before any production change, check the health of the system. Disk full? Out of memory? CPU pegged? These commands tell you in seconds.

CommandWhat it showsKey flags
df -hDisk Free — shows disk space per filesystem-h (human readable), -T (show filesystem type)
du -sh *Disk Usage — shows size of each file/dir-s (summary), -h (human), --max-depth=1
free -hRAM and swap usage-h (human), -m (MB), -s 2 (refresh every 2s)
topLive process monitor — CPU, memory, load avgPress q to quit, k to kill, M to sort by mem
htopBetter interactive process viewer (install first)F5 (tree view), F9 (kill), F10 (quit)
uptimeHow long system has been running + load averages(no flags needed)
whoamiCurrent logged-in usernameid (shows uid, gid, groups)
hostnameSystem's hostname-I (show all IP addresses)
 
# Quick system health check — run these in sequence
df -h          # Is disk space OK?
free -h        # Is memory OK?
uptime         # What is the load average?
whoami         # Am I root or a regular user?

# Find what is eating disk space
du -sh /var/log/*  | sort -rh | head -10

⑤ Common Mistakes · 5 min
⚠ Don't Do This!
Mistake #1 — The rm -rf Catastrophe
✗ WRONG: rm -rf / # Deletes EVERYTHING — entire filesystem! # Even with --no-preserve-root protection, this is career-ending
✓ RIGHT: pwd # Check where you are FIRST ls -la # See what will be deleted rm -rf /path/to/specific/dir # Use absolute path
Mistake #2 — Silent Overwrites with cp
✗ WRONG: cp new-config.conf /etc/app/config.conf # Silently overwrites the existing file with NO warning!
✓ RIGHT: cp -i new-config.conf /etc/app/config.conf # -i asks "overwrite?" before replacing. Use always while learning.
Mistake #3 — ls Without Flags (Missing Hidden Files)
✗ WRONG: ls # Shows only visible files. Misses .env, .bashrc, .ssh/, .gitignore
✓ RIGHT: ls -la # Shows ALL files including hidden, with permissions and sizes # You'll catch .env with secrets, .ssh with key permissions, etc.
⑥ Interview Questions · 5 min
🎤 Crack the Interview
1
What does pwd stand for and when do you use it?
▼
pwd stands for Print Working Directory. It displays the absolute path of the directory you are currently in. You should use it constantly — especially before running destructive commands like rm, before writing to a file, and after cd-ing around. In shell scripts, it helps validate that the script is executing from the expected location. Pro tip: combine with PS1 customization to always show your current directory in the prompt.
2
What is the difference between rm and rm -rf?
▼
rm file deletes a single file and prompts if the file is write-protected. rm -rf dir uses two flags: -r (recursive — deletes directories and all their contents) and -f (force — ignores non-existent files, never prompts). Together they are extremely powerful and equally dangerous. Best practice: always specify the exact path, never use wildcards with rm -rf, and always run pwd + ls first to confirm your location.
3
How do you find files larger than 100MB on a Linux system?
▼
Use the find command with the -size flag: 
find / -size +100M -type f 2>/dev/null
Breaking this down: / = search from root, -size +100M = larger than 100 megabytes, -type f = files only (not dirs), 2>/dev/null = suppress permission-denied errors. To also sort by size: pipe to du -sh {} \; | sort -rh.
4
What is the difference between a hard link and a soft link?
▼
A hard link (ln src dst) creates another directory entry pointing to the same inode (the actual data on disk). If you delete the original file, the data still exists via the hard link. Hard links cannot cross filesystems and cannot link directories. A soft link / symlink (ln -s src dst) creates a special file that contains a path to the target — like a shortcut. If the original is deleted, the symlink becomes a dangling link. Symlinks can cross filesystems and can point to directories. In DevOps, symlinks are common for config management (e.g., /etc/nginx/sites-enabled/ pointing to sites-available/).

⚡ Quick Quiz

Which command shows disk space usage for each mounted filesystem?
du -sh
df -h
free -h
top
Answer Key:

df -h — disk free per filesystem. du -sh — disk used by a specific directory. free -h — RAM usage. top — live process monitor. Know all four cold for interviews!
